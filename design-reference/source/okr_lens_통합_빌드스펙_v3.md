# OKR Lens — 통합 빌드 스펙 (R1 · R2 · R3 단일 문서) v3.2

> **2026-06-30** | `new_spec_all.md`(v2.0) + 피평가자(R1) 초안 + 평가자(R2) 정의서 + 인사담당자(R3) 정의서(v1.2)를 **하나로 통합·정합**한 최종 빌드 스펙.
>
> **제품 한 줄**: 2026 OKR 기반 개인평가제도를 처음 운영하는 IT 조직을 돕는 3역할 도구 — KR을 **잘 쓰게(R1) → 잘 검토·승인하게(R2) → 전사 품질·위험을 보고 표준을 키우게(R3)**.
>
> **이 문서의 목적**: 그대로 Claude(코드 빌드)에 입력해 3역할을 병렬 개발하기 위한 **단일 진실 소스**. 추측이 필요 없도록 쟁점을 §B에서 모두 확정했다.

## A. 이 문서 사용법 (빌드 지시)

1. **빌드 순서**: §D **공통(SHARED) 먼저** → §E R1·R2·R3 **병렬** → §G 통합 게이트. 공통 스키마/엔진/criteria가 서지 않으면 역할 작업을 시작하지 않는다.
2. **우선순위 태그 준수**: 각 기능에 `[MVP]` `[lite]` `[P2]` `[P3]`. 7/15 데모는 `[MVP]`+`[lite]`만으로 완결되게 한다.
3. **결정 준수**: §B 의사결정 로그가 코드 규약보다 우선한다. 충돌 시 §B를 따른다.
4. **일정 가정**: Day1 ≈ 7/1, 데모 ≈ 7/15(약 2주). 날짜가 다르면 §G를 그 비율로 스케일.

### 확정된 범위 전제 (간단 결정 묶음)
- Objective는 **선택 입력**, 상위목표 **정렬 없음**(팀 OKR 미정의).
- 멀티 AI 교차검증(Claude/GPT/Gemini)은 **P2**, MVP는 **단일 Claude**.
- R1 전임자 OKR 열람은 **P2**(평가기준·결과 미공개 원칙).
- R1 글쓰기 서포트(말투 다듬기·문장 만들기)는 **`[lite]`**.
- R1 작년 대비 상향/유지 의견은 history 시드 시 **`[lite]`**.
- 인증·권한은 **프로토타입 단일 계정** — Supabase RLS는 개방형(또는 서버측 service role)으로 두고 **사용자별 인증·RLS는 P2**. 알림은 **앱 내 배지/리스트**.
- **데이터 보안**: 프로토타입(Supabase·Vercel 모두 외부 클라우드)에는 **더미 데이터만**. 실 인사데이터의 외부 클라우드 반입은 별도 보안 검토 사항.

## D. 공통 설계 (SHARED) — 누가 · 언제 · 어떻게 ★

> **핵심 전환**: 진단 규칙은 *코드*가 아니라 R3가 관리하는 *데이터(`criteria` 테이블)*다. R3가 기준을 바꾸면 R1 코칭·R2 검증이 함께 바뀐다. 공통 산출물은 **아키텍트 1인이 초안, 3인 PR 승인** 후에만 병렬 작업 시작.

### D-3. 진단 엔진 (`shared/diagnosticEngine.js`) — 룰 코어 + LLM 보조 (D6)
- **룰(결정적, R2·R3 공유)**: `deriveRisk(checks, criteria)` → {score, level, types} · `tagIssues(kr, checks, criteria)` → C-태그 · `classifyJobGroup(kr, criteria)` → 업무군(§E-3 (가) 규칙).

### D-4. Taxonomy (상수, criteria/types에 고정) (D8)
- **업무군(9)**: 개발/요건 · 데이터/정합성 · 자동화 · 성능/튜닝 · 장애/운영안정 · 정시/납기/처리율 · 매뉴얼/표준/문서 · 보안/권한/개인정보 · 고객/사업기여/제안. (`경계`/`미분류` 허용, `기타/복합` 금지.)

## H. 성공 기준 (MVP — 7/15)

**공통/통합**
- [ ] `/` 랜딩 → 3역할 진입. `npm run dev` 한 번으로 3역할 구동.
- [ ] `criteria`(DB) 편집 → 엔진 주입 동작(항목 수 가변), R1/R2 즉시 반영.
- [ ] **R1 제출 → R2 승인 → R3 인사이트 반영** 연쇄.

**R1** — [ ] 유형선택→기초입력(2채널)→대화형 정제→KR유형→상세수립(O/KR/측정/S~D)→단일모델 검토→수정→제출(`approval_status=제출`).
**R2** — [ ] 카드 6종+필터 / [ ] AI Validation N항목 자동입력·수정·위험도 실시간 재계산 / [ ] 작년 OKR 팝업 / [ ] 반려이력 타임라인 / [ ] 승인·반려(AI 초안)·조정요청.
**R3** — [ ] 기준 관리 뷰가 R1/R2 구동 시연 / [ ] 인사이트 3종(난이도 분포·위험유형·사후조정 비중) / [ ] 사례카드 / [ ] 라이브러리 컨셉 1장(방향성).

## I. 시연 시나리오 (10분)

1. **랜딩(0.5분)**: 역할 선택 → 세 화면 구조 소개.
2. **R1(2.5분)**: 입력 → KR 초안·대화형 정제 → 상세수립 → 제출.
3. **R2(2.5분)**: 제출 KR 조회 → AI Validation(자동판정→수정→위험도 재계산) → 작년 팝업 → 승인.
4. **R3(3.5분)**: 인사이트 3종(본부별 난이도·위험유형·**사후조정 비중**) → 사례카드 → 기준 한 줄 바꾸면 (DB 저장·즉시 반영) R1/R2가 바뀌는 루프 → **라이브러리는 "이렇게 표준을 쌓아갑니다" 방향성 1장**.
5. **마무리(1분)**: R1→R2→R3 데이터 한 바퀴 + 소송 안전 톤 강조.

## J. 제외 범위 (Phase 2 / 3)

- **Phase 2 (평가 도우미)**: 연말 증빙 제출·정합성 검증, 멀티 AI 교차검증, 전임자 OKR 열람, R2 코칭 캘린더 풀기능·집중코칭 연동, R3 11시트 내보내기·등급분포 시뮬·라이브러리 수집/큐레이션/배포 구현, **사용자별 인증·RLS, 운영 배포 시 데이터 보안 검토**.
- **Phase 3 (활용 도우미)**: 최종 리포트 자동 생성, 이의제기 인사 데이터, 라이브러리 성과 변별력 피드백, 비지도 업무군 자동 군집.
- **공통 제외**: 실 인사시스템 연동, 실시간 랭킹/이모지 등급, 이메일/슬랙 알림.
