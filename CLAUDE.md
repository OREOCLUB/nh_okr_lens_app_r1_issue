## 작업 기록 규칙

- 세션을 마무리할 때(사용자가 "오늘 여기까지", "정리하자", "끝" 등을 말하면) 반드시 PROGRESS.md를 업데이트한다.
- 큰 작업 단위(화면 하나 완성, 버그 수정 완료, verifier 통과 등)가 끝날 때마다 PROGRESS.md에 중간 기록을 남긴다.
- PROGRESS.md는 최신 날짜가 맨 위에 오도록 역순으로 작성한다.
- 기록 형식:
  - **완료**: 이번 세션에서 끝낸 작업 (파일/화면 단위로 구체적으로)
  - **진행 중**: 시작했지만 끝내지 못한 작업과 현재 상태 (어느 파일의 어느 부분까지 했는지)
  - **다음 할 일**: 이어서 해야 할 작업, 우선순위 순
  - **이슈/결정사항**: 미해결 버그, 보류한 결정, 나중에 다시 볼 것
- 새 세션 시작 시 PROGRESS.md가 존재하면 먼저 읽고, 최신 항목의 "다음 할 일"을 기준으로 작업을 제안한다.
