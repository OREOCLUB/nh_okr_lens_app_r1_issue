## 개발 필수 규칙 (모든 화면 공통)

- 모든 버튼은 클릭 이벤트를 구현한다. 미구현 버튼을 남기지 않는다.
- 모든 입력창은 Enter 키 이벤트를 검토한다.
- 로그인 화면을 제외한 모든 화면은 세션 체크를 한다.
- 삭제 기능은 반드시 Confirm 팝업을 거친다.
- 필수 입력값은 Validation을 구현한다.
- 목록 화면은 페이징과 정렬을 기본 적용한다.
- 모든 API 호출은 성공/실패 처리를 구현한다 (로딩 상태 포함).
- 모든 UI 문구는 코칭 톤을 사용한다. "위반", "오류", "잘못됨" 같은 지적성 표현 금지.
- 평가 규칙은 코드에 하드코딩하지 않고 Supabase criteria 테이블에서 읽는다.
- 면책 조항은 KR 문장에 섞지 않고 전용 필드로 분리한다.
