## 3. 발견 사항 — 개선 필요 (심각도순)

### 🔴 중요 (기능·데이터 정합)

**[REV-01] 조정요청이 R1에게 "반려"로 전달됨**
- 위치: `src/lib/dataAccess.ts:288` (`saveReviewDecision`)
- 현상: okrs.status가 approved/rejected 2값뿐이라 조정요청(adjustment)을 `rejected`로 매핑. R1 화면에서 조정요청과 반려가 구분되지 않음.
- 영향: D3 연쇄의 정보 손실. R1이 "전체 재작성(반려)"과 "일부 보완(조정)"을 구분 못 함.
- **[확정 Q1] 보류.** 근본 해결(okrs.status에 adjustment 추가 또는 R1이 okr_submissions 참조)은 R1 타입(`mockData.ts:47`)·화면(`r1/page.tsx:12-14`) 수정이 불가피 → R1 사이클로 이관. 이번 수정 범위에서 제외.

**[REV-02] 이미 승인된 팀원도 재처리 가능 + 좌측 리스트와 단절**
- 위치: `src/app/r2/page.tsx:183` (모든 상태의 행이 검토로 이동) + `review/page.tsx:195-203`
- 현상: 대시보드에서 승인/작성중 팀원 행 클릭 → 검토 화면에 해당 팀원이 뜨지만, 좌측 "검토 대상" 리스트(결재/반려/조정만)에는 없어 선택 하이라이트가 없음. 승인 건을 다시 반려 처리하는 것도 가능.
- **[확정 Q2] 재처리 금지.** 승인(approved) 상태 팀원 선택 시: 우측 처리 패널의 처리 방향 선택·처리 버튼·임시 저장 비활성 + "이미 승인된 OKR이에요. 열람만 가능해요 🙂" 코칭 톤 안내. AI 검토 실행은 열람 참고용으로 허용해도 무방(선택). 수정 파일: `r2/review/page.tsx` (R1/R3 무관).

**[FB-01] 피드백 전송이 "최근 보낸 피드백"에 반영 안 됨** *(사용자 발견, 기보고)*
- 위치: `src/app/r2/feedback/page.tsx:15-19, 100, 108`
- 현상: 전송 버튼이 성공 alert만 표시, 저장 경로 없음. 목록은 하드코딩 `SENT` 상수.
- 분류: D5 동결 범위 미구현 — 단 성공 alert가 저장을 암시해 오인 유발.
- **[확정 Q3] (a′) localStorage 반영.** 전송 시 `{to, tone, date, preview}`를 localStorage(예: `okrlens_r2_feedback_sent`)에 prepend 저장하고, 화면 로드 시 저장분 + 기존 `SENT` 더미를 합쳐 렌더링. 새로고침·메뉴 이동 후에도 유지(같은 브라우저 한정 — 데모 충분). DB·테이블 신설 없음. 수정 파일: `r2/feedback/page.tsx` 1개 (R1/R3 무관).
